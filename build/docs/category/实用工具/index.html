实用工具

这里是关于 Remix 实用工具的文档。

📄️ Cookies

一个cookie是服务器在HTTP响应中发送给用户的小块信息，用户的浏览器会在后续请求中将其发送回去。这种技术是许多互动网站的基本构建块，它增加了状态，使您能够构建身份验证（参见sessions）、购物车、用户偏好以及许多其他需要记住“登录”用户的功能。

📄️ createRemixStub

此工具允许您通过设置模拟的路由集来单元测试依赖于 Remix hooks/components 的组件：

📄️ 不稳定数据

这是一个与 Single Fetch 一起使用的工具，用于返回原始数据以及状态代码或自定义响应头。这样可以避免将数据序列化为 Response 实例以提供自定义状态/头信息。这通常是对使用 json 或 defer 的 loader/action 函数的替代。

📄️ defer

要开始使用流数据，请查看 Streaming Guide。

📄️ isRouteErrorResponse

此工具仅仅是 React Router 的 isRouteErrorResponse 的重新导出。

📄️ json

这是创建 application/json 响应的快捷方式。它假定您使用的是 utf-8 编码。

📄️ unstable_parseMultipartFormData

允许您处理应用程序的多部分表单（文件上传）。

📄️ 重定向

这是发送 30x 响应的快捷方式。

📄️ redirectDocument

这是一个小包装器，围绕 redirect，它将触发文档级重定向到新位置，而不是客户端导航。

📄️ 替换

这是一个围绕 redirect 的小封装，它将使用 history.replaceState 而不是 history.pushState 触发客户端重定向到新位置。

📄️ 会话

会话是网站的重要组成部分，它允许服务器识别来自同一用户的请求，特别是在进行服务器端表单验证或页面上没有 JavaScript 时。会话是许多允许用户“登录”的网站的基本构建块，包括社交、电子商务、商业和教育网站。

📄️ unstable_createFileUploadHandler

一个 Node.js 上传处理程序，将带有文件名的部分写入磁盘以避免占用内存，未带文件名的部分将不会被解析。应与另一个上传处理程序组合使用。

📄️ unstable_createMemoryUploadHandler
