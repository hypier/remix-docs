钩子

了解我们提供的各种钩子函数及其用法

📄️ useActionData

返回最近路由 action 的序列化数据，如果没有则返回 undefined。此钩子仅返回上下文中路由的操作数据 - 它无法访问其他父路由或子路由的数据。

📄️ useAsyncError

返回最近的 `` 组件的拒绝值。

📄️ useAsyncValue

返回来自最近的 `` 祖先组件的已解析数据。

📄️ useBeforeUnload

这个 hook 只是围绕 window.beforeunload 的一个辅助工具。

📄️ useBlocker

useBlocker 钩子允许您阻止用户离开当前页面，并向他们展示自定义 UI，以便确认导航。

📄️ useFetcher

用于在不进行导航的情况下与服务器交互的钩子。

📄️ useFetchers

返回一个所有正在进行的 fetchers 的数组。这对于应用中没有创建 fetchers 但希望使用其提交以参与乐观 UI 的组件非常有用。

📄️ useFormAction

解析组件层次结构中最接近的路由的 URL，而不是应用程序的当前 URL。

📄️ useHref

根据当前的位置解析完整的 URL，以用作 href 到 link。如果提供了相对路径，它将解析为完整的 URL。

📄️ useLoaderData

返回来自最近的路由 loader 的序列化数据。

📄️ useLocation

返回当前的位置信息对象。

📄️ useMatches

返回页面上当前的路由匹配。这对于使用当前路由创建布局抽象非常有用。

📄️ useNavigate

useNavigate 钩子返回一个函数，允许您根据用户交互或效果在浏览器中以编程方式导航。

📄️ useNavigationType

返回用户到达当前位置时使用的导航类型。

📄️ useNavigation

此钩子提供有关待处理页面导航的信息。

📄️ useOutletContext

在React Context之上的便利 API，返回最近的父级 `` 组件的上下文值。

📄️ useOutlet

返回此路由层级的子路由元素。此钩子由 `` 内部使用，以渲染子路由。

📄️ useParams

返回一个包含当前 URL 中由路由匹配的动态参数的键/值对对象。子路由会继承其父路由的所有参数。

📄️ unstable_usePrompt

unstable_usePrompt 钩子允许您在离开当前页面之前通过 window.confirm 提示用户确认。

📄️ useResolvedPath

解析给定 to 值的 pathname，并将其与当前位置信息的 pathname 进行比较，返回一个 Path 对象。

📄️ useRevalidator

在正常数据变更之外的原因下重新验证页面上的数据，例如窗口聚焦或定时轮询。

📄️ useRouteError

访问在action、loader或渲染过程中抛出的错误，以便在ErrorBoundary中使用。

📄️ useRouteLoaderData

根据 ID 返回给定路由的加载器数据。

📄️ useSearchParams

返回当前 URL 的 searchParams 元组和一个更新它们的函数。设置搜索参数会导致导航。

📄️ useSubmit

`` 的命令式版本，让你，程序员，可以代替用户提交表单。

📄️ unstable_useViewTransitionState

此钩子在指定位置有一个活动的 View Transition 时返回 true。这可以用于对元素应用更细致的样式，以进一步自定义视图过渡。这要求通过 Link（或 Form、NavLink、navigate 或 submit 调用）为给定导航启用视图过渡。
